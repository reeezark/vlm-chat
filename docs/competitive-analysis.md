## 一、功能对比

| 功能维度 | 本项目 | 成熟竞品 | 差距 |
|---------|--------|----------|------|
| 多模态输入 | 单张图片 + 文本 | 多图、PDF/文档、视频、音频、文件 | 🔴 大 |
| 流式输出 | ❌ 阻塞式 | ✅ 全部支持 | 🔴 大 |
| 模型选择 | 固定 Qwen-VL-Plus（仅环境变量切换） | UI 内多模型/多厂商热切换 | 🟠 中 |
| 会话管理 | 新建/切换/删除/清空 | + 重命名、搜索、置顶、文件夹、分享 | 🟠 中 |
| 提示词/角色 | ❌ | System Prompt、角色市场、Agent | 🔴 大 |
| 工具调用 | ❌ | 联网搜索、代码执行、Function Call、MCP | 🔴 大 |
| 知识库/RAG | ❌ | 文档问答、向量检索 | 🟠 中 |
| 输出渲染 | 纯文本（Gradio Markdown） | Markdown/代码高亮/LaTeX/Mermaid/表格 | 🟠 中 |
| 消息操作 | ❌ | 重新生成、编辑、复制、删除、引用 | 🟠 中 |
| 多语言 UI | 仅中文 | i18n 多语言 | 🟡 小 |

**核心缺失**：流式输出、工具调用/联网、System Prompt、多图与文档输入、消息级操作（重生成/编辑）。

## 二、技术对比

| 维度 | 本项目 | 成熟竞品 | 差距 |
|------|--------|----------|------|
| 架构 | 单体 Gradio，前后端耦合 | 前后端分离（Next.js + 服务端）/微服务 | 🟠 中 |
| 并发 | Gradio 默认队列，同步阻塞调用 | 异步 IO、流式 SSE、水平扩展 | 🔴 大 |
| 存储 | 本地 JSON 文件（每次全量重写） | DB（Postgres/SQLite/Redis）+ 对象存储 | 🔴 大 |
| 状态管理 | 进程内存（已修复多用户隔离） | 无状态服务 + 外部会话存储，可多副本 | 🟠 中 |
| 性能 | 历史图重复 base64、无缓存、首字延迟高 | 流式、缓存、CDN、图片懒加载 | 🔴 大 |
| 安全 | 已加基础鉴权；无限流/审计/内容安全 | OAuth/SSO、速率限制、内容审核、密钥托管 | 🟠 中 |
| 可扩展性 | 模型与渲染硬编码，扩展需改源码 | 插件体系、Provider 抽象、配置化 | 🟠 中 |
| 可观测性 | 仅本地 logging | 指标/链路追踪/用量统计/错误上报 | 🟠 中 |
| 代码质量 | 模块清晰但耦合、测试覆盖局部 | 完整 CI、类型检查、高覆盖 | 🟡 小 |

**关键短板**：同步阻塞 + 文件存储，决定了无法支撑真实并发与水平扩展。

## 六、差距汇总（按影响程度排序）

### 🔴 P0 关键差距（决定可用性/竞争力下限，应优先）

1. **无流式输出** — 体验代差最直接，竞品标配。改造成本低（`stream=True` + generator），收益最高。
2. **同步阻塞 + JSON 文件存储** — 架构性瓶颈，无法支撑并发与水平扩展。需引入异步调用 + DB（SQLite/Postgres）+ 对象存储。
3. **单一模型供应商、无 Provider 抽象** — 锁定 DashScope，扩展性差。应抽象统一 LLM Provider 层。
4. **无工具调用/联网/System Prompt** — 缺失现代助手核心能力，功能层面落后一代。

### 🟠 P1 重要差距（影响体验与生态）

5. 多图 / 文档（PDF）输入缺失。
6. 移动端响应式 + 无障碍 (a11y) 缺位。
7. 消息级操作（重新生成、编辑、复制、停止生成）缺失。
8. 富文本渲染（代码高亮/LaTeX/Mermaid）缺失。
9. 无 Docker 容器化交付，部署门槛高。
10. 无用户/多租户体系与用量计量（商业化前置条件）。

### 🟡 P2 一般差距（锦上添花）

11. 主题仅暗色、缺明暗切换。
12. UI 国际化（i18n）。
13. 文档站、社区运营、LICENSE/贡献规范。
14. 可观测性（指标/用量/错误上报）。

## 七、结论与建议路径

本项目定位是**教学/课程级 VLM 问答 Demo**，模块划分清晰、已修复多用户与安全等基础问题，但与成熟产品存在**一代功能差 + 架构承载差**。

**优先改进顺序建议**：

1. 先补**流式输出**（最高性价比，立刻拉近体验）；
2. 再做**存储层迁移（DB）+ 异步调用**，解锁并发与扩展；
3. 抽象**多 Provider 模型层**；
4. 补齐 **System Prompt + 工具调用** 等核心功能；
5. 最后做容器化、移动端/a11y、社区生态。

其中第 1、2 项是把"可演示 Demo"升级为"可部署产品"的关键门槛。
